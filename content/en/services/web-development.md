Most "web development" pitches are just "I know React". This isn't that. I build the kind of user-facing apps that have real complexity behind them — configurators, dashboards, internal tools, customer portals — the apps that don't quite fit any off-the-shelf product.

## Sounds familiar?

- An off-the-shelf SaaS almost works, but stops at the one thing you actually need
- Your team built something in spreadsheets that's now too painful to live with
- The interface you ship needs to handle real complexity (forms, flows, permissions) without making users miserable

## What you get

- An app that solves your actual problem, not a generic one
- Code the next developer can read without a treasure map
- Performance that isn't an afterthought — fast loads, real device support, accessibility from day one

## How I work

I usually start with the workflow, not the tech. We map out who uses what and where it hurts, then build the smallest thing that proves the core hypothesis. From there we iterate. No 6-month "phase 1" that ships nothing.

I work AI-assisted, but human-led. The decisions that matter — architecture, edge cases, what *not* to build — those are mine.
