Designer verlieren ihren Tag nicht ans "Designen". Sie verlieren ihn ans Exportieren von Assets in vier Größen, ans Token-Syncing, ans Suchen, welche Komponente *eigentlich* zum Design System gehört, und ans Copy-Paste von Texten aus einer Übersetzungstabelle.

Ein gutes Plugin macht diesen Schmerz verschwinden.

## Kennst du das?

- Eure Designer machen das gleiche 5-Klick-Ritual hundertmal pro Woche
- Design Tokens in Figma und Tokens im Code laufen jedes Quartal auseinander
- Ihr braucht Figma, das mit einem CMS, einer internen API oder einer Content-Quelle spricht — und es gibt keine Brücke von der Stange

## Was sich ändert

- Ein Plugin, das zum spezifischen Workflow eures Teams passt — kein generisches Tool, das umgebogen wurde
- Stunden pro Designer, pro Woche, zurückgewonnen — bei den langweiligen Sachen
- Optional: eine OAuth-Integration zu euren internen Systemen, damit Figma sieht, was euer Produkt sieht

## Wie ich arbeite

Ich schaue dem Workflow zuerst über die Schulter. Die meisten Plugin-Anfragen starten als "wir wollen einen Button, der X macht" — und entpuppen sich nach einem Tag Mitschauen als ganz anderes Problem. Also gucke ich zuerst, baue dann.
